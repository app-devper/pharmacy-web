## บทบาท (Role) และสิทธิ์

| Role | สิทธิ์โดยสรุป |
|------|---|
| **SUPER** | ทุกอย่าง |
| **ADMIN** | จัดการยา, ปรับสต็อก, ตรวจนับสต็อก, พิมพ์ฉลากบาร์โค้ด, นำเข้า, ดูรายงาน, ยกเลิกบิล, ตั้งค่าร้าน, แบบฟอร์ม ขย., จัดการผู้ใช้ |
| **USER** | ขาย, ดูสต็อก, ดูประวัติ, เพิ่ม/แก้ลูกค้า, คืนยา |

Role จะแสดง badge ที่ Topbar (มุมขวาบน) ทุกหน้า

## เข้าสู่ระบบ

1. เปิดเว็บ → หน้า Login
2. กรอก **username** + **password**
3. ถ้าถูกต้อง → เข้าระบบและ redirect ไปหน้าขาย
4. Token มีอายุ → ระบบ keep-alive ต่ออายุทุก 10 นาทีโดยอัตโนมัติ
5. ออกจากระบบ: คลิกชื่อมุมขวาบน → **ออกจากระบบ**

> ถ้า session หมดอายุขณะใช้งาน → ระบบจะ redirect ไปหน้า login อัตโนมัติและแจ้ง "กรุณาเข้าสู่ระบบใหม่"

## หน้าขาย — Flow พื้นฐาน

หน้าขาย = หน้าหลักของ cashier · ซ้าย = ช่องค้นหา + กริดยา · ขวา = ตะกร้า

### 1. ค้นยา
- **พิมพ์ในช่องค้นหา** — ค้นจากชื่อการค้า / ชื่อสามัญ / เลขทะเบียน
- **สแกนบาร์โค้ด** (USB HID scanner) — เครื่องสแกนส่ง input เหมือน keyboard · ระบบจับอัตโนมัติ
- บาร์โค้ดรองรับทั้ง barcode หลักของยา + barcode ของ alt-unit (แผง/กล่อง) + เลขทะเบียน (reg_no) เป็น fallback

### 2. เพิ่มลงตะกร้า
- **คลิกที่ card ยา** → เพิ่ม 1 หน่วยหลัก (เช่น 1 เม็ด)
- **ถ้ายามีหน่วยทางเลือก** (เช่น "แผง = 10 เม็ด") → คลิก card จะเปิด dropdown ให้เลือกหน่วย
- **สแกน barcode ของหน่วยทางเลือก** → เพิ่มในหน่วยนั้นโดยอัตโนมัติ

### 3. ปรับจำนวน
- **ปุ่ม + / -** ใน cart row → เพิ่ม/ลดทีละ 1 (ใน "หน่วยที่เลือก")
- **พิมพ์จำนวน** ในช่อง qty ได้โดยตรง
- ยาเดียวกัน แต่หน่วยต่างกัน = คนละ row

### 4. รับเงิน + ออกใบเสร็จ
- กรอก **"รับเงิน"** (หรือคลิก quick-cash chip: พอดี / 100 / 500 / 1000)
- ระบบคำนวณเงินทอนอัตโนมัติ
- กด **"ออกใบเสร็จ"** (หรือ Enter)
- ใบเสร็จขึ้นมา → พิมพ์ด้วยเครื่องพิมพ์ 58/80mm

## หน้าขาย — หน่วยขายและราคา

### หน่วยทางเลือก (Alt units)
- ยาหลักเก็บในหน่วย base (เช่น "เม็ด")
- admin กำหนดหน่วยทางเลือก เช่น "1 แผง = 10 เม็ด" · ราคาขาย/แผง = 100 บาท
- ตอนขาย cashier เลือกหน่วยไหนก็ได้ · ระบบแปลงเป็นหน่วยหลักก่อนตัดสต็อกอัตโนมัติ
- แต่ละ alt-unit สามารถมี barcode ของตัวเอง + hide ไม่ให้โชว์ในหน้าขายได้

### Multi-tier pricing (ราคาตามประเภทลูกค้า)
| Tier | คำอธิบาย |
|------|---|
| `retail` (หน้าร้าน) | ราคาขายปกติ — default |
| `regular` (ลูกค้าประจำ) | ราคาพิเศษลูกค้าประจำ |
| `wholesale` (ขายส่ง) | ราคาส่ง |
| custom (VIP, staff, ...) | admin ตั้งเพิ่มได้ |

**การทำงาน:**
- ลูกค้าแต่ละคนตั้ง tier ได้ในหน้า "ลูกค้า"
- เลือกลูกค้าในตะกร้า → ราคาเปลี่ยนเป็น tier นั้นทันที (ทุก item ในตะกร้า)
- ถ้า alt-unit/ยาไม่ได้ตั้ง tier → fallback retail → fallback `sell_price` ของยา

- ทั้งสองประเภทแสดงรวมใน "ยอดสุทธิ"

## หน้าขาย — ขายล่วงหน้า (Oversell)

เมื่อสต็อกในระบบแสดง 0 หรือน้อยกว่าที่ลูกค้าต้องการ แต่ร้านต้องการขายก่อน:

### Flow
1. เพิ่มยาลงตะกร้า (qty เกิน stock ได้) · card ยาที่ stock ≤ 0 จะ**แสดง badge "ขายล่วงหน้า"** สีส้ม
2. กด "ออกใบเสร็จ" → modal สีส้มแจ้ง **"ยืนยันขายล่วงหน้า"** — แสดง:
   - ยาแต่ละตัว · ต้องการ vs มีในสต็อก · ส่วนเกิน
   - checkbox **"ทราบและยืนยันขายล่วงหน้า"**
3. ติ๊ก checkbox + กด **"ยืนยันขายเกิน"** → บิลออก · stock ของยาไป "ติดลบ"

### Reconciliation
เมื่อร้านนำเข้ายาใหม่ (Import) ล็อตต่อไป → ระบบจะ **auto-reconcile** อัตโนมัติ:
- ตัด "หนี้สต็อก" ออกจากล็อตใหม่ตามลำดับเวลาขาย (เก่าที่สุดก่อน)
- อัพเดต `lot_splits` ย้อนหลัง — pharmacist ตรวจสอบล็อตได้ครบ

### Reconciliation ผ่าน Stock Adjustment
admin ปรับสต็อกเป็นบวก (เช่น นับเจอของ) → ระบบเติมหนี้ oversold ก่อน:
- Adjust +10 กับ stock=-7 → stock=+3, oversold=0, marker `ADJUST:<เหตุผล>`
- Adjust +3 กับ stock=-7 → stock=-4, oversold=4 (ยังค้าง)

### ข้อควรรู้
- **คืนยา oversold ไม่ได้** ส่วนที่ยังไม่ reconcile (ไม่มีล็อตจริงรองรับ)
- หน้าสต็อกยา **drug ที่ติดลบ**แสดงเลขสีแดง + badge "ติดลบ · รอเข้าของ"
- หน้าประวัติการขาย **บิลที่มี oversold** แสดง badge "⏳ ขายล่วงหน้า N"

## หน้าขาย — พักบิล (Hold)

รองรับการพักบิลสูงสุด **5 ช่อง**พร้อมกัน · ใช้เมื่อลูกค้าคนเดิมกลับไปหยิบของเพิ่ม หรือสลับลูกค้าระหว่าง cashier

### วิธีใช้
- แถบ **ParkTabs** ด้านล่างจอ → แต่ละช่องเก็บตะกร้า 1 ชุด
- คลิก **"พักบิลปัจจุบัน"** → ย้ายตะกร้าปัจจุบันเก็บไว้ใน slot ว่าง
- คลิก slot ที่เก็บไว้ → ดึงออกมาใช้ต่อ · ตะกร้าปัจจุบันที่ active สลับเก็บโดยอัตโนมัติ
- คลิก × ใน slot → ยกเลิกตะกร้านั้น

> ตะกร้าพักบิลจะคงอยู่ใน browser (localStorage) แม้ปิด browser ยังไม่หาย

## ประวัติการขาย — ยกเลิก / คืนยา

เข้าหน้า **"ประวัติการขาย"** ใน sidebar

### กรอง + ค้นหา
- **ช่วงวันที่** — from/to (default วันนี้)
- **ค้นหา** — เลขที่บิล หรือชื่อลูกค้า
- กดที่ row เพื่อดูรายละเอียด

### ยกเลิกบิล (Void)
⚠️ **ADMIN เท่านั้น**
1. เปิด detail modal ของบิล → ปุ่ม "ยกเลิกบิล"
2. ระบุเหตุผล → ยืนยัน
3. ระบบจะ:
   - คืนสต็อกยาทั้งหมดในบิล (drug.stock + lot.remaining)
   - หัก total_spent ออกจาก customer (ถ้ามี)
   - Mark บิล voided (ไม่ลบ, เก็บเป็นประวัติ)

### คืนยา (Return)
1. เปิด detail modal → ปุ่ม "คืนยา"
2. เลือกรายการที่คืน + ระบุจำนวน (คืนบางส่วนได้)
3. ระบุเหตุผล (เช่น "ลูกค้าเปลี่ยนใจ")
4. บันทึก → ระบบ:
   - คืนสต็อก + lot remaining ตามที่คืน (FEFO reverse)
   - สร้างใบคืน `RET-YYMMDD-NNN` link กับบิลเดิม
   - คืนเงินตาม `refund`

> ยาที่เป็น **oversold** ยังไม่ reconcile — คืนไม่ได้ (ต้องรอ import หรือ adjust ก่อน)

## สต็อกยา

หน้า **"สต็อกยา"** = รายการยาทั้งหมด

### ดูรายการ
- แถว = ยา · คอลัมน์: ชื่อ, ชื่อสามัญ, ขนาด, ประเภท, รายงาน ขย., ทุน, ขาย, คงเหลือ, สถานะ, บาร์โค้ด, จัดการ
- **คลิก row** → เปิดประวัติการปรับสต็อกของยานั้น
- **Stock badge**: ปกติ (เขียว) / ใกล้หมด (เหลือง) / หมด (แดง) / ค้างส่ง (แดง, stock < 0)

### Action per drug
- **แก้ไข** — เปิดหน้าแก้ไข (ข้อมูลทั่วไป · หน่วยนับ+ราคา+หน่วยทางเลือก · แจ้งเตือนสต็อก · ขย.) — ADMIN
- **ล็อต** — modal รายการล็อตเรียงตามวันหมดอายุ ASC (FEFO queue) · เพิ่มล็อตใหม่ / ลบล็อต
- **ปรับสต็อก** — modal ระบุ delta ±N + reason (นับสต็อก / ยาเสียหาย / ...) — ADMIN
- **ประวัติ** — audit log การปรับสต็อก

### เพิ่มยาใหม่
- ปุ่ม "+ เพิ่มยา" → หน้า `AddDrug` · ถ้า stock > 0 **ต้องระบุล็อต**พร้อมกัน (lot number + วันหมดอายุ + qty + ทุน)
- ปุ่ม "Gen เลขล็อต" → สร้าง `LOT-YYMMDD-XXXX` อัตโนมัติ

### Reorder suggestions
- ปุ่ม "แนะนำสั่งซื้อ" → modal แสดงยาที่คำนวณ avg daily sale × lookahead แล้วน้อยกว่า stock
- เลือก + กำหนดจำนวน → สร้างใบนำเข้า draft ตรง ๆ

## ตรวจนับสต็อก

หน้า **"ตรวจนับสต็อก"** (ADMIN) ใช้ทำ cycle count / ตรวจนับจริงเทียบกับระบบ

### วิธีใช้
1. ค้นหายาที่ต้องการตรวจนับ
2. กรอกจำนวนที่นับได้จริงในช่อง **นับได้**
3. ระบบแสดงส่วนต่างเทียบกับ stock ในระบบทันที
4. ใส่หมายเหตุ เช่น `ตรวจนับประจำเดือน`
5. กด **บันทึกรอบตรวจนับ**

### ผลลัพธ์หลังบันทึก
- ระบบสร้างเลขรอบตรวจนับ `SC-YYMMDD-NNN`
- รายการที่มีส่วนต่างจะถูกปรับ `drug.stock` อัตโนมัติ
- ระบบสร้าง audit log ในประวัติปรับสต็อกด้วย reason `นับสต็อก`
- ถ้าปรับเพิ่มและยานั้นมี oversell ค้าง ระบบ reconcile oversold ให้อัตโนมัติ

## พิมพ์ฉลากบาร์โค้ด

หน้า **"พิมพ์ฉลากบาร์โค้ด"** (ADMIN) ใช้พิมพ์ฉลากยาแบบ Code 128 ผ่าน browser print

### วิธีใช้
1. ค้นหาและเลือกยา หรือเพิ่มแถวว่างแล้วเลือกจาก dropdown
2. เลือก lot ถ้าต้องการให้ฉลากแสดง lot/วันหมดอายุ
3. ตรวจ barcode — ถ้ายาไม่มี barcode ระบบจะใช้ drug id เป็นค่าเริ่มต้น
4. ระบุจำนวนดวง
5. เลือกขนาดฉลาก `38×25 มม.` หรือ `50×30 มม.`
6. กด **พิมพ์**

### ข้อมูลบนฉลาก
- ชื่อร้าน
- ชื่อยา
- lot / วันหมดอายุ (ถ้าเลือก lot)
- barcode graphic แบบ Code 128 + ตัวเลข/ข้อความ barcode
- ราคาและหน่วยขาย (ถ้าเปิด checkbox ราคา)

## จัดการวันหมดอายุ

หน้า **"จัดการวันหมดอายุ"** (ADMIN)

### ดู + กรอง
- Filter: 30 / 60 / 90 / 180 วัน · หรือ **หมดอายุแล้ว**
- แสดงล็อตเรียง `days_left` ASC

### Bulk write-off
- เลือก check หลายรายการ → ปุ่ม "ตัดจำหน่าย"
- ระบุเหตุผล → ระบบ $dec drug.stock ต่อล็อต · เก็บ audit
- Export XLSX ของล็อตที่กรองไว้

## นำเข้าสินค้า

หน้า **"นำเข้า"** (ADMIN)

### Flow
1. **สร้างใบนำเข้าใหม่** → status: **draft** (เลขที่ `IMP-YYMMDD-NNN`)
2. กรอก: supplier, invoice_no, receive_date, รายการยา (drug + lot_number + expiry + qty + cost)
3. **บันทึกแบบร่าง** ไว้ก่อนได้ · แก้ไข/ลบ draft ได้
4. **"ยืนยันรับสินค้า"** → ระบบสร้าง DrugLots + `$inc drug.stock` + บันทึก ขย.9 ในธุรกรรมเดียว
5. ยืนยันแล้ว → status: **confirmed** · แก้ไขไม่ได้

### Oversell reconcile อัตโนมัติ
เมื่อ confirm import ที่ drug มี SaleItems `oversold_qty > 0` → ระบบจะ drain หนี้ FIFO ตามลำดับบิล · อัพเดต `lot_splits` ของบิลเก่าย้อนหลัง

### รายการค้างซิงค์
หน้า **"รายการค้างซิงค์"** (`/offline-sync`) ใช้ตรวจรายการ offline queue ในเครื่องนี้

- ดูจำนวนบิลค้างซิงค์ / จำนวนที่ล้มเหลว / มูลค่ารวม
- ดู error ล่าสุดของแต่ละบิล
- กด **ลองซิงค์ทั้งหมด** เพื่อ replay queue
- กด **ยกเลิก** เพื่อลบรายการออกจาก queue ของ browser นี้ (รายการนั้นจะไม่ถูกส่งเข้า backend)

- ขายได้ปกติ · บิลออนไลน์จะเป็นเลขจริง · บิลออฟไลน์ขึ้น `OFFLINE-XXX` ชั่วคราว → sync แล้วเปลี่ยนเป็นเลขจริง

- หน้า report / stock adjust / import confirm **ต้องออนไลน์**
- ถ้าล้าง browser data → บิลที่ค้าง queue จะหาย

## คำถามพบบ่อย (FAQ)

### Q: ขายยาไม่ได้ แสดงว่า "หมด" แต่ในร้านมีของ
A: ใช้ **Oversell** — cashier กด + เพิ่มลงตะกร้าต่อได้ (card จะเป็น "ขายล่วงหน้า") · ตอน checkout จะมี modal ให้ยืนยัน · จากนั้น admin ปรับสต็อกเพิ่มเพื่อ reconcile

### Q: รายงาน "วันนี้" ไม่แสดง order ช่วงเช้ามืด (00:01-07:00)
A: ตรวจ **Timezone** ใน Settings → ข้อมูลร้าน · ต้องตั้งเป็น `Asia/Bangkok` (หรือ TZ ของประเทศ) · ค่า default คือ `Asia/Bangkok`

### Q: scan barcode แล้วหาไม่เจอ
A: ระบบ fallback 3 ระดับ: (1) barcode ของ alt-unit (แผง/กล่อง) → (2) barcode หลักของยา → (3) เลขทะเบียน (reg_no) · ถ้ายังไม่พบ โปรดเช็คว่ายานั้น activate barcode field แล้วหรือยัง

### Q: ลูกค้าคืนยา — ยาที่ขายเป็น "แผง" จะคืนยังไง?
A: ระบบเก็บใน base unit เสมอ (เช่น 1 แผง = 10 เม็ด → SaleItem qty=10) · คืนใน detail modal จะแสดงเป็น **base unit** (เม็ด) · ระบุจำนวนเม็ดที่คืน

### Q: ทำไมบางบิลแสดง badge "⚠ lot drift"?
A: คือบิลที่ขายตอน offline แล้ว sync เมื่อ online · ช่วงนั้น lot FEFO ใน server เปลี่ยน (terminal อื่นขายก่อน) ทำให้ ล็อตที่ระบบตัดจริง ≠ ล็อตที่ client คาดเดาไว้ตอนขาย · ข้อมูลถูกต้องทุกอย่าง แค่ paper trail อาจต้อง cross-check

### Q: พักบิลได้กี่บิล?
A: 5 ช่อง · เก็บใน localStorage ของเครื่องนั้น · กลับมาเปิด browser ยังอยู่

### Q: stock ติดลบ แก้ยังไง?
A: หลายทาง:
1. **Import** — นำเข้ายาใหม่ → ระบบ reconcile อัตโนมัติ
2. **Adjust +N** — admin ปรับสต็อก positive → ระบบชำระหนี้ oversold ให้ก่อน
3. **Void** — ยกเลิกบิลที่ขายเกินสต็อก → คืนทั้งหมด

### Q: ADMIN vs USER ต่างยังไง?
A: USER ขายได้, ดูประวัติ, ดูสต็อก, เพิ่ม/แก้ลูกค้า · ADMIN เพิ่มเติม: จัดการยา, ปรับสต็อก, นำเข้า, รายงานกำไร/EOD, ยกเลิกบิล, ตั้งค่า, ขย.

### Q: ออกใบเสร็จแล้วอยากพิมพ์ใหม่?
A: เข้า **ประวัติการขาย** → คลิก row → ปุ่ม **พิมพ์ใหม่**
